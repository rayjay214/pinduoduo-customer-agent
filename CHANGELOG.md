# 更新记录

## v2026.06.16 - 2026-06-16

### 主要更新

- RAG 预检索改为按订单状态、最新物流状态、客户是否确认收到来判定售前/售中/售后，不再用客户关键词粗略猜大场景。
- 场景内检索改为混合召回和重排：向量召回、关键词/BM25 风格召回、别名/短语精确召回、meta 重排、命中质量闸门。
- 预检索 query 支持结合最近客户上下文，短句追问可以补全主题，同时避免旧主题污染当前问题。
- `search_knowledge` 删除旧商品知识库/旧客服知识库兜底；新场景知识没有可靠命中或缺少商品时，直接代码层转人工。
- 工具转人工结果由 Agent 直接返回固定客户话术，不再交给模型二次生成。
- 知识库 UI 新增商品族管理：商品族列表、售前/售中/售后结构化知识编辑、绑定链接列表。
- 商品族绑定链接页面新增“添加链接”，支持把新同步商品加入已有商品族并生成三场景结构化知识。
- 知识库总页面新增“一键重构知识库索引”。
- 增加多组 RAG、工具、商品族、UI 生命周期和安全约束测试。

### 安全与发布说明

- 本版本不包含 `config.json`、数据库、日志、runtime 分析数据、cookie、token、浏览器登录态或本地 Excel 导入文件。
- 未跟踪压测脚本中发现疑似真实 API Key，已明确排除在本次发布之外。
- 更新线上服务后需要重启进程，Python 代码改动才会生效。

## v2026.06.09 - 2026-06-09

### 主要更新

- 新增 TurnContext 结构化上下文，拆分客户文本、商品卡、订单卡、媒体消息和场景提示。
- 修复商品卡、订单卡元数据污染客户问题的问题。
- 支持价格分转元，例如 1161 解析为 11.61。
- 修复 CustomerAgent 中 `get_config` 未导入导致回复失败的问题。
- 售后场景收到图片或视频时，直接转人工处理。
- 售后场景反馈没有充电口、找不到插电口、在哪里插电时，直接转人工处理。
- 多订单待确认场景默认按售后处理，避免有订单客户被误判为售前。
- 移除图片单独消息在 AIReplyHandler 中提前回复的逻辑，让图片消息进入 CustomerAgent 统一处理。

### 注意事项

- 本版本不包含 runtime 分析数据、聊天记录、日志、cookie、token 或本地缓存文件。
- 更新线上服务后需要重启进程，Python 代码改动才会生效。
